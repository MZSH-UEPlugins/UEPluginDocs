# 📘 SimpleStaticMeshMerger 用户指南

本指南详细介绍如何使用 SimpleStaticMeshMerger 插件合并静态网格体实例，优化场景性能。

---

## 🛠️ 打开工具窗口

启用插件后，在编辑器菜单中找到工具入口：

### 步骤：

1. 打开 Unreal Engine 编辑器
2. 点击顶部菜单 `Window > Static Mesh Merger`
3. 工具窗口将作为可停靠面板打开

---

## 🎨 界面概览

工具窗口由三个主要区域组成：

### 1. 工具栏（顶部）

- **Break Level Instances**：递归拆分场景中所有 LevelInstance（包括嵌套的）
- **Refresh**：手动刷新合并预览
- **Type Dropdown**：选择合并类型（HISM 或 ISM，默认 HISM）
- **Merge Selected**：对选中的分组执行合并操作

### 2. 配置选项（工具栏下方）

#### Mobility 过滤器（Filter）
- ☑ **Static**：包含静态对象
- ☑ **Stationary**：包含固定对象
- ☐ **Movable**：包含可移动对象
- 默认：勾选 Static + Stationary

#### Min Instance Count
- 范围：0 - 1000
- 默认：2（仅显示至少 2 个实例的分组，合并至少需要 2 个可勾选实例）
- 目的：过滤掉实例过少、优化收益极小的分组

### 3. 双面板布局

#### 左面板：合并分组预览
显示所有符合条件的合并分组，每个包含：
- ☑ **复选框**：选择参与合并的分组
- **分组名称**：使用第一个未排除实例的名称（动态更新）
- **实例详情**：显示"实例数 • 类型 • Mobility"（如 `5 Instances • HISM • Static`）

顶部功能：
- **搜索框**：按实例名称模糊搜索（分词 + 顺序匹配）
- **Select All**：批量勾选/取消当前显示的分组

#### 右面板：包含的 Actor
显示左面板选中分组中的所有实例：
- ☑ **复选框**：勾选表示包含在合并中，取消勾选表示排除
- **Actor 名称**：实例在场景中的标签
- **[⊙] 定位按钮**：点击跳转到场景中的 Actor 并在 Outliner 中选中

顶部功能：
- **Toggle All**：批量勾选/取消所有实例

---

## 🔍 搜索功能

### 搜索规则

支持**分词模糊搜索**，关键词必须按顺序出现：

**示例**：搜索 `SM 02`
- ✅ 匹配：`SM_Rock_02`（"SM" 在 "02" 前面）
- ✅ 匹配：`SM_000002`（"SM" 在 "02" 前面）
- ❌ 不匹配：`02_SM_Rock`（顺序错误）
- ❌ 不匹配：`Rock_02`（缺少 "SM"）

### 搜索行为

- **仅影响显示**：不修改数据，清空搜索框恢复所有分组
- **分组级过滤**：如果分组中有任意实例匹配，整个分组显示（包含所有实例）
- **合并限制**：仅合并当前显示且已勾选的分组

---

## ⚙️ 分组规则

插件自动将**属性完全相同**的 Actor 归入同一分组，包括：

1. **静态网格体**：使用的静态网格体资产
2. **Mobility**：Mobility 类型（Static/Stationary/Movable）
3. **完整碰撞设置**：
   - 碰撞启用类型（NoCollision/QueryOnly/PhysicsOnly 等）
   - 碰撞配置文件名
   - 碰撞对象类型
   - 所有 32 个碰撞通道的响应
4. **材质**：材质列表

**只有所有属性完全相同的 Actor 才会被合并到一个分组。**

---

## 🚀 使用工作流

### 基本合并流程

1. **打开工具**：`Window > Static Mesh Merger`
2. **（可选）拆分 LevelInstance**：点击 `Break Level Instances` 按钮
3. **配置过滤器**：
   - 勾选需要的 Mobility 类型
   - 设置最小实例数阈值
4. **查看预览**：左面板显示所有合并分组
5. **（可选）搜索过滤**：在搜索框输入关键词
6. **勾选分组**：
   - 点击 `Select All` 批量勾选
   - 或手动勾选要合并的分组
7. **（可选）排除实例**：
   - 点击左面板分组查看右侧实例列表
   - 取消勾选不想合并的实例
8. **执行合并**：点击 `Merge Selected` 按钮
9. **查看结果**：合并完成后自动选中新创建的 ISM/HISM Actor

### 进阶技巧

#### 精确控制合并对象
1. 使用搜索框缩小范围（如搜索 `Tree` 仅显示树相关分组）
2. 取消勾选不需要的分组
3. 进入右面板排除个别实例
4. 点击合并仅处理当前显示且已勾选的分组

#### 定位场景对象
- 点击右侧实例列表中的 `[⊙]` 按钮
- 编辑器视口跳转到该 Actor
- Outliner 同步选中

#### 自动取消勾选
排除实例导致可勾选数量 < 最小实例数时，分组自动取消勾选

---

## 📊 实例数规则

### 显示规则
- Actor 总数 >= 最小实例数

### 可勾选规则
- 可勾选 Actor 数 >= 最小实例数

### 合并执行
- 实际参与合并的 Actor 数 >= 最小实例数

**示例**（最小实例数 = 2）：
- 分组有 5 个实例
- 取消勾选 3 个实例
- 剩余 2 个可勾选实例
- **可以合并**（2 >= 2）

## 🏗️ 拆分 LevelInstance

### 功能

点击 `Break Level Instances` 递归拆分场景中所有 LevelInstance：

- ✅ 自动处理嵌套 LevelInstance
- ✅ 智能去重（相同路径的重复实例被删除）
- ✅ 实时进度通知（滚动显示当前项目）
- ✅ 每轮迭代后 UI 自动刷新

### 处理策略

| 场景 | 处理方式 | 结果 |
|------|---------|------|
| 首次遇到路径 | 加入缓存 → 拆分 | 内容移至主场景 |
| 同级重复 | 检测 → 删除 | 避免重复内容 |
| 嵌套重复 | 检测 → 删除 | 避免循环引用 |

## ✅ 注意事项

### 合并规则
- 仅合并**主场景（PersistentLevel）**中的 Actor
- 不包含 LevelInstance 内部的 Actor（需先拆分）
- 所有属性必须相同才会归为一组

### 碰撞保留
- 完整复制碰撞配置文件
- 精确复制对象类型和所有 32 个通道响应
- 合并后碰撞行为与原始 Actor 一致

### 最小实例数
- 默认 2：至少需要 2 个实例才能合并
- 过滤掉实例过少的分组（优化收益极小）
- 可按需调整（范围 0-1000）

### 场景监控
- 自动监控 Actor 添加/删除（500ms 延迟后刷新）
- 切换地图时清空列表（需手动点击 Refresh）

### 状态缓存机制
- **排除列表持久化**：取消勾选的实例记录在排除列表中
- **刷新时保持状态**：点击 Refresh 或场景变化自动刷新**保留**排除状态
- **窗口级生命周期**：仅关闭并重新打开窗口时所有实例重置为默认勾选状态
- **使用场景**：可逐步调整排除实例，多次刷新验证结果而无需重复操作

### 合并后
- 自动选中所有新创建的 ISM/HISM Actor
- 视口跳转到第一个新创建的 Actor
- 原始 Actor 被销毁

## 🎯 性能优化建议

### 合并策略

1. **优先合并大量重复**：实例越多，优化效果越好
2. **分批合并**：使用搜索按类别合并（如先合并树，再合并石头）
3. **保留必要分组**：不要合并需要单独控制的对象

### HISM vs ISM

- **HISM**（默认推荐）：
  - 支持 LOD
  - 支持遮挡剔除
  - 大规模实例性能更好

- **ISM**：
  - 更简单轻量
  - 小规模实例开销更低
